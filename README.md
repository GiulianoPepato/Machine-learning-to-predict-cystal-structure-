# Machine-learning-to-predict-cystal-structure-based-on-atomic-features
O presente reposório contém toda a parte técnica (códigos e arquivos html) gerados para realizar a minha iniciação científica orientado pelo professor Gustavo Dalpian e financiada pela CNPq. O tema do projeto é "Usar Machine Learning para predizer a Estrutura Cristalina de Materiais baseado em Features Atômicas", que é motivado pelo fato de que processos de sintetização, caraceterização, cálculos de primeiros principios e de DFT para estimar essa propriedade requerem um alto consumo de recursos materiais, computacionais e financeiros, muito diferente de uma abordagem de ML, que custa muito menos e é muito mais rápida embora possua uma precisão menor. Por esse motivo, é  importante estudar que conjuntos de features baseadas apenas nos elementos químicos que compõem o material e qual algoritmo de Machine Learning que melhor otimiza essa previsão e comparar com demais trabalhados da literatura. Todo o código feito para avaliar esse processo pode ser acessado no arquivo `ML_crystal_structure.ipynb`. Caso queria análisar o artigo, me contate via giuliano.pepato@usp.br.

Para facilitar o acesso e poupar o tempo de quem quiser se aprofundar na análise das limitações, eu já deixei disponibilizado a lista de arquivos html dos grafos e das nuvens de probabilidade. Para mais informações sobre, confira os tópicos abaixo.

## Nuvens de Probabilidade

Imagine um espaço vetorial tridimensional, onde cada dimensão representa o número átomico. Dessa maneira é possível atribuir um material em um ponto nesse espaço discreto dada a sua composição química. Por exemplo: H_2O é a molécula se água que é composta por 2 elmentos, H (Z=1) e O (Z=16), dessa forma, podemos atribuir o ponto (1,16,0) à molécula de água, contudo, algoritmos de ML não sabem química, então temos que fazer com que ele não dê importância para ordem. Para fazer isso não somente atribuimos um ponto no espaço para um certo material, mas sim 6 (permitação de 6 elementos). Com isso, criamos a nossa coleção de pontos/materiais de uma certa estrutura cristalina. Regiões que concentram maiores números de pontos são mais densas, que podem indicar que é mais provável encontrar materiais da estrutura cristalina escolhida naquela região do que nas demais, portanto fazemos representações dessas regiões mais ou menos densas atrávez de uma nuvem de probabilidade nesse espaço. O raciocínio anterior não é baseado em ciências dos materiais, mas sim uma compreensão lógica que algoritmos de ML podem adotar. Por isso, para avaliar algumas tendências de resultados do _Random Forest_, eu criei as nuvens de probabilidade de materiais de cada estrutura tanto para os dados do conjunto de treinamento (NuPr_estrutura) quanto para os dados do conjunto de validação (NuPr_estrutura_val), pois é preciso ver comparar a nuvem com o qual o modelo aprendeu contra a nuvem no qual ele tentou predizer para avaliar se os erros cometidos são justificadas pela semelhança (estatística) entre elas.
